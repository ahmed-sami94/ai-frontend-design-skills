# Section Recipes

Use these recipes to build original frontend pages from the recurring WordPress theme patterns.

## Hero

Best for public sites where visitors need fast orientation.

- Content: category/offer, short value proof, CTA, secondary action, visual asset.
- Layout: split only when the image is directly useful; otherwise use full-bleed or editorial composition.
- States: mobile crop, shorter mobile height, safe text wrapping, accessible CTA focus.
- Avoid: empty gradient hero, vague illustration, overloaded carousel, text hidden inside a card.

## Header

- Minimal: logo, nav, CTA.
- Service business: top contact strip, logo, nav, quote/book CTA.
- Ecommerce: logo, categories, search, account, cart.
- Content site: logo, nav, search, subscribe.
- Dashboard/app: compact shell, account/menu, workspace switcher, current page title.

## Services Or Features

- Use 3-6 items.
- Make each item answer a different user question.
- Prefer icon plus concise label plus one benefit.
- Use equal card dimensions only for repeated comparable items.
- Add a next-step link when the user naturally drills down.

## Trust Band

Choose the strongest available proof:

- Numbers: projects, clients, uptime, years, locations.
- Logos: partners, clients, certifications.
- Reviews: rating, quote, source, name/role.
- Compliance: badges, warranties, guarantees, safety standards.

## Portfolio Or Gallery

- Use strong imagery with visible subject matter.
- Add filters only when there are enough items.
- Keep hover overlays concise.
- Include result, category, location, or date when it helps comparison.

## Team

- Use only when people matter to trust.
- Include role, specialization, credential, or contact action.
- Avoid anonymous decorative headshots.

## Process

- Use 3-5 steps.
- Each step should reduce uncertainty: intake, plan, execute, review, support.
- Show timeline, deliverable, or responsibility if useful.

## Contact Or Booking

- Include a real conversion path: form, phone, map, booking, quote, chat, or download.
- Keep forms short unless the service requires qualification.
- Add reassurance near the form: response time, privacy, consultation, or no-obligation note.

## Ecommerce Product Grid

- Stable card size.
- Product image, title, price, sale state, rating or attribute, quick action.
- Visible cart feedback.
- Filters and sorting when item count is high.

## Footer

- Use 3-5 columns: brand/contact, services/products, company, support/legal, social/newsletter.
- Repeat the main CTA only when it is helpful, not as filler.

## Domain Adaptation

- Healthcare: appointments, practitioners, insurance, departments, emergency/contact clarity.
- Education: courses, admissions, outcomes, events, faculty, downloads.
- Real estate: search/filter, featured listings, neighborhood proof, agent CTA.
- Hospitality: booking widget, rooms/menu, gallery, amenities, location, reviews.
- Nonprofit: cause, impact stats, donation CTA, programs, transparency.
- Agency/software: outcomes, services, case studies, process, pricing or consultation CTA.
- Repair/local service: phone-first header, service area, urgent CTA, warranties, before/after.
